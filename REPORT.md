## 2. Прохождение данных через систему

### Шаг 1: Входные данные (Trigger)

Файл `input_data.json` содержит массив пользователей:

```json
[
  {
    "user_id": "u-9921",
    "first_name": "Александр",
    "last_action": "viewed_pricing",
    "days_inactive": 3,
    "segment": "warm_lead",
    "favorite_feature": "AI-summarization"
  },
  {
    "user_id": "u-4402",
    "first_name": "Мария",
    "last_action": "export_report",
    "days_inactive": 14,
    "segment": "churning_pro",
    "favorite_feature": "API-integration"
  }
]
```

### Шаг 2: Обработка (Processing)

Для каждого пользователя:
1. **System Prompt** задаёт ToV и ограничения (без капса, обращение на «Вы», P.S. по favorite_feature).
2. **User Prompt** подставляет данные: `first_name`, `segment`, `days_inactive`, `last_action`, `favorite_feature`, `user_id`.
3. Логика по сегментам:
   - `warm_lead` → мягкое напоминание о преимуществах после просмотра цен.
   - `churning_pro` → сожаление о перерыве, приглашение вернуться.

### Шаг 3: Валидация

- Subject ≤ 60 символов
- Body: 400–800 символов
- `user_id` в `cta_link` и `unsubscribe_link`
- Наличие CTA в конце письма
- Только разрешённые функции: AI-summarization, API-integration

### Шаг 4: Выход

- `output/generated_emails.json` — полные данные для каждого письма
- `output/report.html` — визуальный отчёт (для скриншотов)

## 5. Демонстрация (скриншоты)

Для записи видео или скриншотов:

1. **Скриншот 1:** Открыть `input_data.json` — показать входные данные.
2. **Скриншот 2:** Запустить `run.bat` или `python main.py` — показать вывод в консоли.
3. **Скриншот 3:** Открыть `output/report.html` в браузере — показать таблицу результатов.
4. **Скриншот 4:** Прокрутить до примеров писем — показать полный текст для Марии и Александра.

**Запуск с OpenAI API (реальная генерация):**
```powershell
$env:OPENAI_API_KEY="sk-..."
python main.py
```

**Запуск в mock-режиме (без API):**
```powershell
$env:CONTENT_GEN_MODE="mock"
python main.py
```
Или просто без ключа — система автоматически переключится на mock.

## 7. Принципы реализации

- **KISS:** Один цикл, один вызов API на пользователя, простая валидация.
- **DRY:** Общие промпты в `prompts.py`, шаблон User Prompt с подстановкой.
- **Best Practice:** Разделение System/User промптов, жёсткий JSON-формат ответа, проверка кодировки для кириллицы.
